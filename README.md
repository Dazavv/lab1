# Работа 1: Разработка защищенного REST API с интеграцией в CI/CD

## Описание проекта

Проект представляет собой REST API для аутентификации пользователей, реализованный на Spring Boot.
Он предоставляет следующие возможности: регистрация пользователей, вход в систему и выдачу JWT-токенов для последующих запросов к защищённым ресурсам.

## Эндпоинты API

### Регистрация пользователя

**POST** `/api/auth/register`
**Тело запроса (JSON):**

```json
{
  "login": "user123",
  "password": "password",
  "name": "John Doe"
}
```

**Ответ (JSON):**

```json
{
  "token": "jwt-token-string"
}
```

### Авторизация пользователя

**POST** `/api/auth/login`
**Тело запроса (JSON):**

```json
{
  "login": "user123",
  "password": "password"
}
```

**Ответ (JSON):**

```json
{
  "token": "jwt-token-string"
}
```

## Реализованные меры безопасности

1. **Защита от XSS**
   Все входные данные, получаемые от пользователя (`login`, `password`, `name`), проходят экранирование с помощью `HtmlUtils.htmlEscape()`, чтобы предотвратить внедрение вредоносного HTML/JS-кода.

2. **Защита от SQL Injection (SQLi)**
   Все операции с базой данных выполняются через Spring Data JPA и подготовленные запросы, что исключает возможность внедрения SQL-кода.

3. **Аутентификация**

   * Для регистрации и входа используется сервис `AuthService`, который возвращает JWT-токен.
   * JWT-токен используется для последующих запросов к защищённым эндпоинтам.
   * Пароли пользователей хранятся в базе данных в зашифрованном виде (BCrypt или аналог).


## CI/CD и SAST/SCA

Проект интегрирован с GitHub Actions для автоматической проверки кода и безопасности:

* **SpotBugs** — статический анализ кода (SAST)
* **Snyk** — проверка зависимостей на уязвимости (SCA)
